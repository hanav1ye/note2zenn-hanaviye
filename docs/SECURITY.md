# セキュリティ

## スコープ

note2zenn は **ローカルで動作する VSCode 拡張**です。次をセキュリティ上のスコープとします。

- 本リポジトリのソースコード（TypeScript、Docker 定義、GitHub Actions ワークフロー）
- 公式ドキュメントに記載された想定どおりの使い方

次は **利用者の運用・環境** に依存するため、本ポリシーの脆弱性対応の主対象外とします（README の「注意事項」参照）。

- VSCode 設定や SecretStorage の誤設定による誤ったディレクトリへの書き込みや `git push`
- OpenAI 等への記事本文送信に伴う組織ポリシー違反
- API 料金・ディスク容量・処理時間の増大
- 利用者が拡張コマンドに意図的に渡した URL への自己起因のアクセス

## サポートするバージョン

| バージョン | サポート |
|------------|----------|
| リポジトリの default ブランチ（現状 `master`） | 対応 |

タグ付きリリースがない場合、最新の default ブランチを対象とします。

## 脆弱性の報告

セキュリティ上の問題を発見した場合は、**公開 Issue ではなく** 次のいずれかで連絡してください。

- リポジトリの **Security Advisories**（GitHub の「Report a vulnerability」）
- リポジトリ管理者が設定している非公開の連絡先

報告時に含めていただきたい情報:

- 再現手順（コマンド、`.env` の該当項目は値をマスクしてよい）
- 影響（例: 意図しないファイル書き込み、シークレットの露出）
- 想定される深刻度と環境（OS、Docker の有無）

## 対応方針

- 内容を確認し、再現可能であれば修正または回避策を検討します
- 重大度が高い場合は、修正が用意できるまでの間、Issue や README での注意喚起を行う場合があります
- 対応期限は保証しませんが、可能な範囲で迅速に対応します

## 利用者向けの推奨事項

- `GITHUB_TOKEN` は **最小権限**（対象リポジトリのみ）の fine-grained PAT または同等に限定する
- 本番の Zenn リポジトリへ push する前に **必ず diff を確認**する
- 社内秘を含む可能性がある原稿を OpenAI API に送信しない
- API キーやトークンを平文ファイルではなく SecretStorage に保存する

詳細な責任範囲と想定利用は [README.md](README.md) の「注意事項・利用上の前提」を参照してください。
