# note2zenn

note 記事を Zenn 向け Markdown に変換し、画像取得・OpenAI リライト・Zenn 用リポジトリへのコピー・`git push`（任意）までを行う **VSCode 拡張**です。

## クイックスタート

1. `npm install && npm run build` のあと F5 で拡張を起動（開発時）
2. 左 Activity Bar の **Note2Zenn** サイドバーを開く
3. 設定を入力して **設定を保存**、秘密情報ボタンで API キー等を登録
4. **note URL** と **basename** を入力し **変換を実行**

## 注意事項・利用上の前提

本ツールは **利用者自身のマシン上で、利用者が指定した URL と設定に基づいて動作する**ことを前提としています。以下を読んだうえで利用してください。

### 想定している使い方

- **ローカルの VSCode 拡張**として実行する
- 変換対象は **自分が内容を把握している note 記事**（原則として本人の記事）
- `note2zenn.zennRepoPath` は **テスト用または意図した Zenn 用リポジトリ** を指す
- `git push` を使う場合は、**生成物（Markdown・画像・コミット内容）を確認してから** 公開する
- 社内秘・個人情報・認証情報を含む原稿を **外部 LLM（OpenAI）に送らない**

### 想定していない使い方

- インターネット上に HTTP サーバとして公開し、第三者に実行させること
- **内容を確認していない記事**（特に他人の note）を、そのまま本番の Zenn リポジトリへ `git push` すること
- CI 上での無人・無確認の自動公開（本ツールの標準的な運用範囲外）
- 社内ポリシーで禁止されているデータを、許可なくクラウド LLM API に送信すること

### 利用者の責任範囲

次は **利用者側の設定・運用・判断** により発生し得る事象であり、本ツールの提供者が結果を保証するものではありません。

| 領域 | 内容 |
|------|------|
| **設定** | VSCode 設定と SecretStorage の値（`zennRepoPath`、Git トークン、OpenAI API キー等）の正しさ |
| **変換元** | 指定した note URL および取得した HTML の信頼性（他人の記事・改ざんされたコンテンツを含む） |
| **公開内容** | LLM の出力、フロントマター、リンク・画像を含む最終的な Markdown の確認と公開判断 |
| **コスト・容量** | OpenAI 等の API 料金、ディスク使用量、処理時間、モデルダウンロード時間 |

| **Git** | `commit` / `push` の対象ブランチ・リモート・履歴への影響 |

### データの送信先（事実の明示）

| 設定 | 送信・保存の行き先 |
|------|-------------------|
| OpenAI 固定 | 記事本文などが **OpenAI の API** へ送られる（利用規約・データ取扱いは OpenAI 側に従う） |
| 記事取得・画像 DL | コマンド入力した URL および HTML 内の画像 URL へ HTTP リクエストが行われる |
| GitHub token 設定時 | HTTPS push のために **GitHub** へ認証情報が用いられる |

### 重大な問題になりうる運用（参考）

次の組み合わせでは、セキュリティインシデントやコンプライアンス上の問題に発展し得ます。**利用者が回避する責任**があります。

1. **他人の note を無確認で本番 Zenn リポジトリへ push** — プロンプト注入等により、意図しない文言・リンクが公開名義で載る可能性
2. **社内秘を含む原稿を OpenAI に送信** — 組織ポリシー上のデータ持ち出しに該当し得る
3. **広い権限の `GITHUB_TOKEN` を漏らす** — 別経路（ログ共有・誤コミット等）と重なった場合、リポジトリへの不正利用につながり得る

本ツール単体がリモートから任意実行される設計ではないため、上記は主に **運用・信頼境界** に関するリスクです。
